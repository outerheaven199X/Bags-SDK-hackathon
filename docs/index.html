BagsSDK MCP Server

The batteries-included MCP server that gives AI agents full access to the Bags.fm API.

40 Tools 4 Resources 6 Prompts Agent Mode

What is this?

bags-sdk-mcp is an MCP (Model Context Protocol) server that exposes the entire Bags.fm API surface as tools, resources, and prompts. Any MCP-compatible AI client — Claude Desktop, Cursor, or custom agents — can interact with the Bags.fm ecosystem through natural language.

Key Features

  • 40 MCP tools — trading, token launches, fee management, claiming, analytics, Dexscreener, and more
  • Fee Config Composer — fluent builder for multi-party fee splits without raw BPS math
  • Tiered caching — 5-level TTL cache to stay within the 1,000 req/hr rate limit
  • Zero-custody — no private keys stored; all transactions returned unsigned as base64
  • Autonomous agent mode — dual-model orchestration with Hermes 4 + Claude Sonnet
  • Dual transport — stdio for Claude Desktop, Streamable HTTP for remote connections

Who is this for?

  • Token creators wanting to launch and manage tokens through conversational AI
  • Fee earners who want to track and claim fees hands-free
  • Partners building on top of Bags.fm's fee infrastructure
  • Developers integrating Bags.fm into AI-powered applications
💡 Built for the Bags.fm Hackathon This server wraps @bagsfm/bags-sdk v1.3.1 and the Bags.fm REST API into a single MCP interface.

Installation

Prerequisites

  • Node.js 18 or later
  • npm 9+
  • A Bags.fm API key — get one at dev.bags.fm

Install from npm

npm install -g bags-sdk-mcp

Install from source

git clone https://github.com/npittman/bags-sdk-mcp.git
cd bags-sdk-mcp
npm install
npm run build

Verify installation

bags-sdk-mcp --help

You should see the CLI usage output with available flags.

Quick Start

Once configured in Claude Desktop, you can start using BagsSDK tools through natural language. Here are some things to try:

Check your portfolio

"What's my portfolio look like? My wallet is ABC123..."

Claude will call bags_wallet_balance, bags_token_holdings, and bags_claimable_positions to give you a complete overview.

Get a swap quote

"How much SOL would I get for selling 1000 tokens of mint XYZ?"

Claude calls bags_quote and shows you the expected output amount and price impact.

Claim your fees

"Claim all my fees above 0.01 SOL"

Claude calls bags_claim_all_fees with a dust threshold, returning unsigned transactions for each position.

Launch a token

"Help me launch a token called MOON with symbol $MOON"

Claude walks you through the full launch flow using the bags_launch_token prompt — metadata upload, fee config, and launch transaction.

⚠️ You always sign transactions yourself Every transaction-generating tool returns unsigned base64 data. You sign with your own wallet (Phantom, Solflare, Backpack) before broadcasting.

Zero-Custody Model

BagsSDK MCP never touches your private keys. The architecture is designed so that:

  • Transaction-generating tools (swap, claim, launch, fee config) return unsigned base64-encoded VersionedTransaction objects
  • You sign externally using your wallet (Phantom, Solflare, Backpack)
  • You broadcast using bags_send_transaction after signing

Transaction Flow

┌──────────────┐    ┌───────────┐    ┌──────────┐    ┌──────────┐
│ AI requests  │ →  │ BagsSDK   │ →  │ You sign │ →  │ Broadcast│
│ bags_swap    │    │ returns   │    │ with     │    │ via      │
│              │    │ unsigned  │    │ Phantom  │    │ send_tx  │
│              │    │ base64 tx │    │          │    │          │
└──────────────┘    └───────────┘    └──────────┘    └──────────┘
🔒 Security guarantee No private key, seed phrase, or signing capability ever passes through this server for standard operations. The bags_agent_wallet_export tool (agent mode only) can export agent wallet keys — handle with extreme care.

Tiered Caching

The Bags.fm API has a 1,000 requests/hour rate limit. The cache layer protects against hitting it with five TTL tiers:

TierTTLWhat Gets Cached
ImmutableForeverSocial → wallet resolution (never changes)
Stable10 minCreators, pool configs, partner configs
Moderate5 minLifetime fees, claim stats, token feed, pools
Volatile2 minClaimable positions
None0Quotes, transactions, auth, send-tx

The cache is in-memory (Map-based) with no external dependencies. Entries auto-expire on read. Prefix-based invalidation is available for bulk cache clearing.

💡 Real-time data is never cached Quotes, transaction building, and on-chain broadcasts always hit the API directly to ensure freshness.

Error Handling

Every tool wraps its handler in try/catch and maps HTTP status codes to actionable messages:

StatusMessage
401Check your BAGS_API_KEY. Get one at dev.bags.fm
429Rate limited. BagsSDK caches responses. Retry in 60 seconds.
404Resource not found. Verify addresses/mints are correct.
Connection errorCannot reach Bags API. Check network and BAGS_API_BASE.

Errors are returned as MCP isError: true responses, so the AI client can explain what went wrong and suggest fixes.

Static vs Template Resources

Static resources have a fixed URI and return the same data structure each time (though the data refreshes based on cache TTL).

Template resources accept a parameter in the URI path (like a token mint or wallet address) and fetch data dynamically. They use ResourceTemplate from the MCP SDK.

Example: Reading a token resource
// AI client reads bags://token/EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v
// Returns composite data: pool info + creators + lifetime fees

Fee Config Composer

The FeeConfigBuilder provides a fluent API for composing fee split configurations without dealing with raw BPS math.

Fluent API

TypeScript
FeeConfigBuilder.create()
  .addRecipient("twitter", "alice", 5000)    // 50%
  .addRecipient("twitter", "bob", 3000)      // 30%
  .addDividendsBot(2000)                      // 20% to holders
  .validate()  // { valid: true, errors: [] }

Template Presets

TemplateDescriptionDefault Split
soloCreator100% to one person10000 BPS
creatorPlusDividendsCreator + DividendsBot50/50
teamSplitEven split across membersAuto-calculated
influencerLaunchCreator + influencers + dividends30% / split / 20%
daoDAO treasury + dividendsConfigurable

Validation Rules

  • At least 1 recipient, max 100
  • All BPS must be positive integers
  • BPS must sum to exactly 10000 (100%)
  • No duplicate provider:username pairs
  • Warns when >15 claimers (needs lookup tables)

Using via MCP Tool

The bags_compose_fee_config tool exposes this as either:

  • mode: "custom" — manually specify recipients and BPS
  • mode: "template" — use a preset template name

It returns the validated config without submitting to the API, so you can review before committing.

Agent Mode

Activated with the --agent flag. Runs autonomous strategies using dual-model AI orchestration.

⚠️ Requires additional API keys Agent mode needs NOUS_API_KEY (Hermes 4) and ANTHROPIC_API_KEY (Claude Sonnet) in your environment.

Dual-Model Routing

The orchestrator routes tasks based on complexity using regex pattern matching:

ModelHandlesTrigger Patterns
Hermes 4 (fast)Routine tasksclaim, balance, check, list, monitor, status
Claude Sonnet (strategic)Complex decisionsevaluate, analyze, recommend, optimize, compare, strategy

Available Strategies

auto-claim

Checks positions every 5 minutes. Claims fees above 0.01 SOL threshold. Logs unsigned transactions for external signing.

bags-sdk-mcp --agent --auto-claim

launch-monitor

Watches the launch feed every 30 seconds. Tracks seen mints to avoid duplicates. Escalates interesting launches to Sonnet for analysis.

bags-sdk-mcp --agent --monitor

Combined

bags-sdk-mcp --agent --auto-claim --monitor

Programmatic-Only Strategies

These are available via code but not exposed as CLI flags:

  • portfolio-rebalance — sends position summary to Sonnet for rebalancing suggestions
  • fee-optimizer — sends fee configs to Sonnet for optimization suggestions
